## 本周精选

- **有人真做出了汤姆·里德尔的日记**——本周新奇与传播双高（novelty 9.0 / viral 9.0）：开发者 Maxime Rivest 开源 riddle，在 reMarkable 墨水屏上写字，停笔约 2.8 秒墨迹被「纸」吸走，AI 用流动的手写体一笔一划写回来；工程由 Claude Fable 5 完成，HN 608 分登顶、外媒跟进。值得写：没有聊天框、纸笔本身成了 AI 的界面，文化+体验向非技术读者也能看懂。
- **ChatGPT 语音整个换了引擎**——GPT-Live 全双工上线（viral 8.5）：AI 第一次学会「边听边说」和「不抢话」，复杂问题在后台委派 GPT-5.5 边聊边查，即日起全球推送、覆盖每周超 1.5 亿语音用户。值得写：用「打电话 vs 对讲机」的比喻把三代语音架构讲给普通读者，或做插话、停顿、背景噪音三场景实测。
- **你视频下的一条评论，可能正在指挥 YouTube 官方 AI**——本周最该敲警钟的信号（novelty 8.5 / viral 8.5）：安全研究员实测 YouTube Studio 的 AI 助手 Ask Studio 被评论区存储型 prompt 注入「策反」，还能钓走私密视频标题，Google 回应「不算 bug」。值得写：给所有平台创作者的自保清单——别轻信后台 AI 输出里的「官方通知」、别乱点它给的链接。
- **公众号排版终于「一键化」了**——8 天 1542 星（accessible 9.5，本周上手门槛最低）：两位公众号作者联名开源 gzh-design-skill，让任何 coding agent 把 Markdown 排成可直接粘贴、不掉格式的公众号 HTML。值得写：拿自己一篇旧文重排做前后对比，素材就是自己的文章，公众号博主人人可写。
- **18 个词、30 秒一个、错一个出局**——13 小时拿下 HN 836 分的 Show HN（viral 8.5）：极简每日文字游戏 18 Words 再次验证「每日一题 + 限时 + 晒成绩」的 Wordle 公式在 2026 年依然能造爆款。值得写：拆解小游戏的传播机制，或盘点「不太会代码也能用 AI 做每日小游戏」的路径。
- **一个人用 1300 行 C，把 744B 模型塞进 25GB 内存的笔记本**——本周最新奇之一（novelty 9.0）：colibrì 纯 C 零依赖，int4 的 GLM-5.2 常驻内存仅 9.9GB，370GB 权重放硬盘按需流式读取，Apple M5 Max 实测 1.06 tok/s。值得写：用「饿着养活大模型」的比喻讲清 MoE 按需加载 + 磁盘流式，慢但真能跑，离线与隐私是卖点。
- **23 年前的《命令与征服：将军》被搬上 iPhone**——「人导演 + AI 工程师」的另一实证（novelty 8.0 / viral 8.0）：设计师 Ammaar Reshi 开源移植项目，真引擎编译到 ARM64、配触屏 RTS 操作，工程由 Claude Code 完成，未删改的工程日志全公开。值得写：精读 PORTING_PLAYBOOK.md，提炼「指挥 AI 干大活」的方法论；情怀向给当年玩《将军》《红警》的一代也能写。
- **21 万 star 的不是 agent，是「配置」**——Everything Claude Code 把 Claude Code/Cursor/Codex/OpenCode 统一成一层「agent 操作系统」（novelty 8.0）：一份 AGENTS.md 跨工具复用技能、记忆与安全规则，还配开源审计器 AgentShield 扫配置里的提示注入。值得写：当模型和工具快速换代，「配置资产」正在成为新基础设施，写给已在用 coding agent 的读者。
- **世界上大多数人能摸到的 AI，不是 ChatGPT**——IEEE Spectrum 特稿冲上 HN（accessible 8.5）：最穷国家只有 0.7% 网民用过 ChatGPT（发达国家约 25%），假药鉴别、心电图、病虫害识别靠的是手机和 50 美元单板机里的离线小模型。值得写：AI 报道里少见的「离你很远但真实发生」的一面，用 0.7% vs 25% 的认知反差落笔。
- **12 年前的老 CPU 也能跑的免费配音**——82M 参数的开源 TTS Kokoro 被实测「真能用了」（accessible 9.0）：纯 CPU 4.7 秒合成接近真人的语音、支持中文，一条 docker 命令起服务、兼容 OpenAI speech API。值得写：视频/播客创作者的省钱账本——和云配音订阅摆一起算账，重点实测中文效果并诚实讲短板。

## 这周怎么用

本周的暗线有两条：一是「一个人 + 一个 AI」开始交付过去要一个团队的作品——riddle 日记和《将军》移植前后脚登顶 HN，且都公开了完整工程日志；二是 AI 在往「更小、更旧、更本地」的设备里钻——colibrì 用硬盘喂 744B、Kokoro 让老 CPU 变配音棚、IEEE 记录小模型在弱网世界干活。如果只做一件事，建议这样排主线：

- 主线一（趋势长文）：精读 riddle 与《将军》两份公开工程日志，写「人当导演、AI 当工程师」的分工方法论，落点是普通创作者如何指挥 AI 干成大活。
- 主线二（实测教程）：「平民本地 AI」一条龙——Kokoro 免费配音 + colibrì 跑前沿模型 + IEEE 案例做背景，做成「AI 正在掉进旧电脑里」的图文或视频，实用与传播兼顾。
- 主线三（安全 + 实操双拼）：先用 Ask Studio 注入案例写一份创作者自保清单（转化快、人人相关）；再用 gzh-design-skill 做一期公众号排版实测，是本周最容易出流量的实操角度。
