# 一起搞着玩

搞着玩是一个文件驱动的内容站：**一篇文章或一张选题卡，就是一个 Markdown 文件**。所以任何人都能参与——加一个文件、提一个 PR 就行，不用懂整套代码。

## 你可以投稿什么

- **选题卡**：一条值得中文创作者写的全球信号。放到 `content/topic-cards/<英文-slug>.md`。
- **文章**：一篇全球信号解读或方法论。放到 `content/posts/<英文-slug>.md`。
- **工具**：往工具库加一个真实工具。编辑 `lib/content.ts` 里的 `tools` 数组。

模板在 `content/templates/`，格式说明见 `content/README.md`。

## 投稿流程

1. Fork 本仓库。
2. 复制对应模板，新建一个 `.md` 文件，按格式填好。
3. 本地跑校验：
   ```bash
   npm install
   npm run check:content   # 内容格式
   npm run typecheck       # 类型
   npm run lint            # 代码规范
   ```
4. 提交 Pull Request。CI 会自动跑上面的检查，绿了才能合并。

## 投稿红线（很重要）

- **只写真实的事**：选题卡和文章必须基于真实来源，`materials` / `sources` 填可访问的真实链接。
- **不编造数据**：热度数字、功能、引用都要能核对。
- **给得出角度**：选题卡至少 3 个写作角度、3 个标题；文章至少 3 个 Pro 角度、3 个备选标题。
- **标清边界**：涉及国内可用性、合规、医疗/法律/金融的，写清楚事实边界，没法核实就别发。

## 不想用 Git？

直接开一个 Issue，把你的选题、来源链接和想法贴上来；维护者会帮你转成选题卡。

搞着玩，就是大家一起搞。
